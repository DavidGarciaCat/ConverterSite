It broke! Why?

<< Back to the converter

  1. The number one reason is that you didn't paste in a services config. This site does not work for routing.yml, config.yml or travis.yml files. It only works for your service configuration: those files inside your Resources/config/services.(xml|yml|php).

  2. If you can convert a file but not visualize it or dump it to Graphviz, then you've likely been bitten by this issue which is trying to be fixed via a patch in Symfony itself. In the meantime, replace any unresolved parameters in your config with a placeholder value and then you can convert. This ONLY applies to Graphviz, other formats should work fine.

  3. Another common error is not including the config header. When converting a XML file, it must start with 

    <container xmlns="http://symfony.com/schema/dic/services"
        xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
        xsi:schemaLocation="http://symfony.com/schema/dic/services http://symfony.com/schema/dic/services/services-1.0.xsd">
    For a YML file, it must start with the "services" key 
    services:
    my_first_service:
    # etc, etc

  4. If you've made it this far, I'm stumped. Double check that the correct format is selected and if the problem persists, please do file a bug report!
